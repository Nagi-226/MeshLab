### 插件功能设计
- **插件名**：filter_normal_enhance（法线可视化增强）
- **功能1**：顶点法线可视化（RGB颜色编码法线方向）
- **功能2**：法线翻转（翻转所有顶点和面法线）
- **参数**：RichBool(重算法线) + RichPercentage(箭头长度) + RichEnum(颜色模式)

### 关键技术点
- 类继承链：QObject → MeshLabPlugin → MeshLabPluginLogger → FilterPlugin → FilterNormalEnhance
- 三个必须的Qt宏：Q_OBJECT + MESHLAB_PLUGIN_IID_EXPORTER + Q_INTERFACES
- 文件末尾必须有：MESHLAB_PLUGIN_NAME_EXPORTER
- 插件注册：在src/CMakeLists.txt的MESHLAB_PLUGINS列表中添加路径

## M1 开发环境搭建 - 已验证通过

### 确认的开发环境：
- **Qt**: E:\Qt\5.15.2\msvc2019_64（Qt5OpenGL/Qt5Xml/Qt5Network 三个模块齐全）
- **Visual Studio**: E:\VS2026（版本 18.4.11626.88，MSVC 19.50.35728，v144 + v145 工具集）
- **CMake**: 3.29.0（C:\Program Files\CMake\bin\cmake.exe）
- **Ninja**: 1.13.2（用于替代 VS 生成器，因为 CMake 3.29 不认识 VS 2026）
- **MeshLab 版本**: 2025.07（单精度浮点）

### 关键发现：
- CMake 3.29 最高只支持到 "Visual Studio 17 2022"，不支持 "Visual Studio 18 2026"
- **解决方案**：用 Ninja 生成器替代 VS 生成器，需要先激活 VS DevShell 环境
- VS2026 的 MSVC v144/v145 编译器可以正常编译链接 Qt5.15.2 msvc2019_64 的库（向下兼容）

### 编译验证结果：
- filter_sample 插件编译成功，生成 filter_sample.dll（40,960 bytes）
- 构建输出位置：build_m1_test/src/distrib/plugins/filter_sample.dll

### 标准构建命令：
```powershell
# 1. 激活 VS2026 开发者环境
Import-Module "E:\VS2026\Common7\Tools\Microsoft.VisualStudio.DevShell.dll"
Enter-VsDevShell -VsInstallPath "E:\VS2026" -DevCmdArguments "-arch=amd64 -host_arch=amd64" -SkipAutomaticLocation

# 2. CMake 配置
cd "E:\Github Project\MeshLab\MeshLab-src"
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_PREFIX_PATH="E:\Qt\5.15.2\msvc2019_64"

# 3. 编译
cmake --build build --target filter_sample
```

### 注意事项：
- winget 已安装新版 CMake（可能在 C:\Program Files\CMake），但需重启终端生效
- CMake 配置中有若干 Warning（embree、TBB、levmar 版本过旧），均为非致命警告
- MESHLAB_BUILD_ONLY_LIBRARIES=ON 可以跳过 meshlab 主程序构建，只编译库和插件
